<task type="auto">
  <name>Task 2: Colored folder shadows</name>
  <files>src/pages/index.astro</files>
  <action>
LOCKED DECISION (CONTEXT.md line 60): "Colored shadows ONLY — NO black shadows. Each folder's shadow matches its section color."

Audit the current folder hover/glow effects in index.astro. Find any box-shadow or drop-shadow that uses black, rgba(0,0,0,...), or an uncolored value. Replace all with colored shadows using each folder's section color.

APPROACH — use CSS `filter: drop-shadow()` on the .folder element (or its SVG), tinted with the section color:

For each folder, the shadow color must match its section color at ~30% opacity:
- Now (blue #3B82F6): `drop-shadow(0 4px 12px color-mix(in srgb, #3B82F6 30%, transparent))`
- Work (green #22C55E): `drop-shadow(0 4px 12px color-mix(in srgb, #22C55E 30%, transparent))`
- Guest Book (amber #F59E0B): `drop-shadow(0 4px 12px color-mix(in srgb, #F59E0B 30%, transparent))`
- About (purple #8B5CF6): `drop-shadow(0 4px 12px color-mix(in srgb, #8B5CF6 30%, transparent))`

IMPLEMENTATION:
1. Read the existing .folder and folder hover CSS in index.astro to understand the current shadow/glow state.
2. Each folder link (`.folder-now`, `.folder-work`, `.folder-guestbook`, `.folder-about`) should get its own hover rule with a colored drop-shadow, for example:

```css
.folder-now:hover .folder-icon {
  filter: drop-shadow(0 4px 16px color-mix(in srgb, #3B82F6 35%, transparent));
}
.folder-work:hover .folder-icon {
  filter: drop-shadow(0 4px 16px color-mix(in srgb, #22C55E 35%, transparent));
}
.folder-guestbook:hover .folder-icon {
  filter: drop-shadow(0 4px 16px color-mix(in srgb, #F59E0B 35%, transparent));
}
.folder-about:hover .folder-icon {
  filter: drop-shadow(0 4px 16px color-mix(in srgb, #8B5CF6 35%, transparent));
}
```

3. If the folders already have a generic `.folder:hover` drop-shadow that's black or uncolored, remove it. The per-folder colored rules replace it.

4. If the folder icons already use a CSS variable (e.g., `--folder-color`) per folder, use that: `drop-shadow(0 4px 16px color-mix(in srgb, var(--folder-color) 35%, transparent))`.

5. Transition: ensure `filter` is included in the transition property for smooth glow on hover. `transition: transform var(--duration-fast) ease-in-out, filter var(--duration-fast) ease-in-out;`

DO NOT add box-shadows (colored or otherwise) — use CSS filter: drop-shadow() only so it follows the SVG shape.
