Inline script (in index.astro <script> tag) for swipe and dot navigation:
```javascript
const carousel = document.getElementById('folder-carousel');
const dots = document.querySelectorAll('.dot');
let current = 0;
let startX = 0;

function goTo(index) {
  current = Math.max(0, Math.min(3, index));
  const folders = carousel.querySelectorAll('.folder');
  folders[current].scrollIntoView({ behavior: 'smooth', block: 'nearest', inline: 'center' });
  dots.forEach((d, i) => d.classList.toggle('active', i === current));
}

// Touch swipe
carousel.addEventListener('touchstart', (e) => { startX = e.touches[0].clientX; }, { passive: true });
carousel.addEventListener('touchend', (e) => {
  const diff = startX - e.changedTouches[0].clientX;
  if (Math.abs(diff) > 40) goTo(current + (diff > 0 ? 1 : -1));
}, { passive: true });

// Dot clicks
dots.forEach((dot, i) => dot.addEventListener('click', () => goTo(i)));
```

IMPORTANT: The folder-carousel-wrapper must be in the HTML alongside the folder-grid. The CSS media queries control which one is visible. Do NOT duplicate the folder data (SVG + labels). Keep it DRY — consider rendering both from the same data array or just have both present in HTML and toggling with CSS.

SIMPLEST APPROACH: Render the folder items ONCE inside .folder-grid. Inside .folder-grid, wrap in a CSS-snap scroll container on mobile. This avoids duplication.

REVISED APPROACH (simpler, no duplication):
- .folder-grid on desktop: flex row with gap
- .folder-grid on mobile: `overflow-x: auto; scroll-snap-type: x mandatory; -webkit-overflow-scrolling: touch;` — native scroll snap, no JS needed for basic swipe
- Each .folder: `scroll-snap-align: start; flex: 0 0 auto;`
- Dots: add .carousel-dots div BELOW .folder-grid, hidden on desktop, visible on mobile. Dots update via JS touchend listener.

USE THIS REVISED APPROACH — it's simpler and doesn't duplicate HTML.

Additional CSS for .home on mobile: `.home` should use `display: flex; flex-direction: column; align-items: center; gap: var(--space-xl);` on mobile (not justify-content: center, so content starts near top with scroll).

Modify .home media query at ≤640px:

The carousel likely uses arrow buttons and dots for navigation. Add touch swipe support:

1. In the React component, add touch state:
```tsx
const [touchStart, setTouchStart] = useState<number | null>(null);

const handleTouchStart = (e: React.TouchEvent) => {
  setTouchStart(e.touches[0].clientX);
};

const handleTouchEnd = (e: React.TouchEvent) => {
  if (touchStart === null) return;
  const diff = touchStart - e.changedTouches[0].clientX;
  if (Math.abs(diff) > 40) {
    if (diff > 0) goNext();
    else goPrev();
  }
  setTouchStart(null);
};
```

<div
  className="carousel-track"
